Overview

This project is to automate the Sprint activations process for customer care, referred to as 'One-button Push'.  Sprint upgrades are the highest activation type for our Costco.com site right now and automating these order activations will increase efficiencies for customer care.

 

Goals

We will be automating all three types in a phased approach (if needed) - Upgrade, New, AAL is the order of priority.  This will be implemented for both Costco and AAFES.

  1. Upgrade Cold Activation One-button push automation deployed in end of September release
    New Cold Activation One-button push automation deployed in end of September release
  2. AAL Cold Activation One-button push automation deployed in end of September release

Requirements

  • Need to implement additional APIs needed to automatically activate an order
  • The activation APIs are implemented in OMT (Order Management Tool used by Care) 
  • Sprint API List we have in production today and the required APIs to implement (also available on Sprint APIs page)
  • All APIs to implement listed below per activation type

Overview of Activation Process - Upgrade Cold

These are the high-level steps of what actions are taken and what the api should be sending to carrier, but there needs to be analysis of the api documentation given to us by the carrier to understand what pieces to implement.  Also, reviewing code from the other carriers in OMT for upgrade activations that are automated will help as well.

  1. Customer places order on our site
  2. Order flows to our OMT (Order Management Tool)
  3. Care pulls up order in OMT and pushes "Activate All" button
  4. The behind-the-scenes api should update the carrier with:
    1. The plan change and/or any additional features, which they update customer's account with
    2. New 2 yr service agreement
    3. Device details are sent to carrier (phone imei/sim) to update on their side
  5. Pushing the button also adds a note in OMT on our side to ship the phone

Required APIs:

  • 7UL/6ZN ACCOUNT_VALIDATION_REQUEST / OrderValidationModule.doAccountValidation
  • 7UL/6ZN CREDIT_CHECK_REQUEST / OrderValidationModule.doCreditCheck

  • 7UL/76S QueryPlansAndOptionsService.validatePlansAndOptionsV1

  • 7UL/76S DiscountManagementService.lockRebateEligibilityV1
  • 7UL/6ZN ACTIVATION_REQUEST / OrderValidationModule.doActivation
  • 7UL/6ZN ACTIVATE_RESERVED_DEVICE_REQUEST / OrderValidationModule.doActivateReservedDevice

Overview of Activation Process - New Cold Activation

A new cold activation is defined as a new Sprint customer creating a new account with a new subscription in which the phone is shipped to the customer un-activated.

  1. Customer places order on our site
  2. Order flows to our OMT (Order Management Tool) 
  3. Care pulls up order in OMT and pushes "Activate All" button

 Required APIs:

  • 7UL/76S QueryCsaService.queryCsaV2
  • 7UL/76S QueryNumberPortingService.checkNumberPortingEligibilityV1
    • Required if porting.
  • 7UL/6ZN CREDIT_CHECK_REQUEST / OrderValidationModule.doCreditCheck
  • 7UL/76S QueryPlansAndOptionsService.validatePlansAndOptionsV1
  • 7UL/6ZN ACTIVATION_REQUEST / OrderValidationModule.doActivation
  • 7UL/6ZN ACTIVATE_NOW_REQUEST / OrderValidationModule.doActivateNow or call customer care
    • Required if doActivation call is using a future dated activation date and now the customer desires to activate before future dated activation date (e.g. shipment arrives early).
